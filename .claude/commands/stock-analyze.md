# 个股 V2+V12 分析命令

对指定股票进行 V2 估值 + V12 技术评分分析，生成独立报告文档。

## 参数

$ARGUMENTS - 股票代码（如 HK.01093）或多个代码逗号分隔（如 HK.01093,HK.01772）

## 执行步骤

### 1. 确保数据最新

同步目标股票的 K 线数据（最近 120 天）。

### 2. V2 估值分析

通过 Web Search 获取最新基本面数据（每只股票最多 1 次搜索，合并关键词如 "[名称] forward PE PB ROE EPS growth 2026"）。

根据股票类型选择指标路径：

**成长股（AI/半导体/新能源/医药）** → Forward PE + PEG
- PEG = Forward PE ÷ 预期EPS增速(%)
- < 0.75 显著低估 / 0.75-1.0 值得买入 / 1.0-1.5 合理 / 1.5-2.0 偏贵 / > 2.0 回避
- A股对卖方增速预期打 8 折

**周期/价值股（矿业/化工/银行）** → PB-ROE + PB/ROE
- PB/ROE = PB ÷ ROE(%)
- < 0.10 深度低估 / 0.10-0.15 优秀 / 0.15-0.20 合理 / 0.20-0.30 偏贵 / > 0.30 回避

**未盈利公司** → 标注"无法评估"，给出 PS、毛利率等替代指标

### 3. V12 技术评分

运行 `python scripts/v12_scoring.py` 获取评分，或从数据库 K 线手动计算：

- OBV 评分（满分 50 → 折算 6 分）
- VCP 评分（满分 50 → 折算 6 分）
- 周 K 修正（±0.5~1 分）
- 信号判定：≥9 极强 / 7-9 强势 / 5-7 中性 / 3-5 转弱 / <3 弱势

### 4. 综合判断

结合 V2 + V12 给出操作建议：
- V2 通过 + V12 ≥ 7：买入信号
- V2 通过 + V12 5-7：关注，等技术确认
- V2 通过 + V12 < 5：观望
- V2 不通过：不参与（无论技术面多好）

### 5. 输出报告

报告保存至 `output/个股分析/[代码]-V12V2分析-[YYYYMMDD].md`

如果 `output/个股分析/` 目录不存在则创建。

## 注意事项

- 估值筛选不通过 → 跳过详细技术评分，直接给结论
- 每只股票 Web Search 最多 1 次
- A 股 PEG 计算对卖方增速打 8 折
- 周期底部不看 PE，只看 PB
- 增速为负不算 PEG，用 PB-ROE 替代
