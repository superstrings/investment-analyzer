# 企稳了 — 交易策略 (Stabilized Strategy)

## 触发条件

执行此 skill 之前，**必须**先运行：

```bash
python scripts/stabilization_check.py
```

得分 ≥ 3/8 才进入此策略；< 3 则继续等待。

## 策略目标

在市场从系统性下跌中刚刚企稳的窗口内，捕捉**反弹波**：
1. **以小搏大 Call 期权**（高赔率小仓位）
2. **分批建仓核心标的**（中线持有）
3. **明确风控**（防止假企稳变二次下跌）

## 执行步骤

### Step 1: 同步数据 + 复算 V12

```bash
python main.py sync all -u dyson
python scripts/v12_scoring.py
```

确认核心 5 标的当前 V12 评分：

| 标的 | 必须满足 |
|---|---|
| AVGO | V12 ≥ 10 + 站上 400 |
| NVDA | V12 ≥ 10 + 站上 MA10 |
| TER  | V12 ≥ 9  + PEG ≤ 1.2 |
| GLW  | V12 ≥ 9  + 站上 MA20 |
| INTC | V12 ≥ 8  + Q1 反转确认 |

**任何一只 V12 跌破阈值 → 该标的不操作；至少 3 只满足才整体启动策略**

---

### Step 2: 候选标的筛选

按以下顺序排查 V12 ≥ 9 的候选标的（结合 4-28 时数据）：

#### A. 一线候选 (Tier 1) — 期权 + 现货同步建仓
- **AVGO** 博通 [V12=12] — 满分核心
- **NVDA** 英伟达 [V12=11] — 距高点最近，弹性最大
- **TER**  泰瑞达 [V12=11, PEG≈1.0] — 估值最佳

#### B. 二线候选 (Tier 2) — 仅小仓位 Call
- **GLW** 康宁 [V12=11] — Q1 业绩催化
- **INTC** 英特尔 [V12=10] — Q1 +29x EPS 反转
- **MRVL** 迈威尔 [V12=10, ⭐持仓] — 不加 Call（已持现货）

#### C. 不操作 (Skip)
- **CIEN** [V12=11 但 Forward PE 82, 距分析师目标 -50%] — 估值太贵
- **COHR** [V12=9 但弱势] — 行业内最弱
- **AMD** [V12=10 但近日大涨] — 等回调

---

### Step 3: Call 期权选择（基于过去失败教训）

**铁律（基于 2026-04-09 LITE/COHR 复盘）**：
- ❌ SC-V12 < 5 不做
- ❌ 日线 OBV < 25 不做
- ❌ 不追开盘暴涨
- ✅ 每只股票最多一个 Call 仓位
- ✅ 日线 OBV 强 + 盘中回调 = 买点

**Call 选择参数**：

| 参数 | 推荐值 | 理由 |
|---|---|---|
| **到期日** | 45-60 DTE | 避开 30 DTE 内 Theta 衰减加速 |
| **行权价** | OTM 5-10% | Delta 0.30-0.40，杠杆与赔率平衡 |
| **单标的仓位** | ≤ 总资金 2% | "以小搏大"原则，小亏可控 |
| **总 Call 仓位** | ≤ 总资金 6% | 即使全部清零也只损失 6% |
| **入场条件** | 日线 OBV 突破前高 + 收阳 K 线 + 量比 > 1 | 三重验证 |
| **止损** | Call 跌 -50% 或标的跌破 MA20 | 严格机械化 |
| **止盈** | Call 涨 +100% 减半仓 | 锁定利润 |

**Call 优先级**（基于 V12 + 历史波动率 + IV 性价比）：

1. 🥇 **AVGO Call**（OTM 5%，Delta 0.40，60 DTE）
2. 🥈 **NVDA Call**（OTM 8%，Delta 0.35，45 DTE）
3. 🥉 **TER Call**（OTM 7%，Delta 0.35，60 DTE）

### Step 4: 现货分批建仓（中线）

**只对持仓 < 5% 仓位的标的考虑加仓**：

| 标的 | 第一批 | 第二批 | 第三批 |
|---|---|---|---|
| **TER** 泰瑞达 | 现价附近 30% | -3% 加 30% | -5% 加 40% |
| **GLW** 康宁 | $160 (MA20) 附近 | $150 加仓 | $145 满仓 |
| **沪电股份 (002463)** | ¥100 附近 | ¥95 加仓 | ¥90 满仓 |

**总仓位上限**：
- 任一标的 ≤ 总资金 5%
- 全市场仓位 ≤ 总资金 70%（保留 30% 应对二次下跌）

---

### Step 5: 风控复核（执行前最后检查）

每笔下单前，逐项确认：

- [ ] **杠杆**：当前杠杆 < 1.5x（持仓杠杆 + Call 杠杆名义价值）
- [ ] **止损**：每笔交易必须有明确止损价（写在执行单里）
- [ ] **仓位**：单标的不超过 5%（现货）+ 2%（期权）
- [ ] **集中度**：同板块（光通信/算力/AI）总仓位 ≤ 25%
- [ ] **对冲**：保留 ≥ 30% 现金 / 货币基金
- [ ] **Q1 业绩**：是否在业绩窗口内（避免业绩前/后48小时建仓）

---

### Step 6: 进场后跟踪

每天收盘后（无论涨跌）执行：

```bash
python scripts/stabilization_check.py
```

**触发条件回退**：

- 若 stabilization_check 得分回落到 < 3 → **立即清仓所有 Call**
- 若任一持仓跌破 MA20 → **立即止损现货**
- 若大盘 SPX 单日跌幅 > 2% → **当日停止任何加仓**

## 历史教训（必读）

### 2026-04-09 LITE/COHR 失败复盘
- **问题**：在系统性下跌时强行建仓 Call，被 IV crush + 方向错误双重打击
- **教训**：必须等企稳信号确认才操作
- **新规**：日线 OBV < 25 不做、SC-V12 < 5 不做、不追开盘暴涨

### 2026-04-28 思格新能止损执行
- **问题**：止损单是消息提醒，不是真正条件单
- **教训**：止损必须是真条件单（STOP_MARKET 优于 STOP_LIMIT 防跳空）
